Befunge

前言

  • befunge是一种特别的编程语言，此题写出了一种befunge的实现，最后通过边界上的溢出进行攻击
  • 可以看到XCTF或者强网很多题都是libc2.31这种低版本，很容易联想到打free_hook，并且可以发现题目里面一般都有free函数。再往高版本显然不可能打堆的IO_FILE，那么就只能是往栈溢出方面想了
  • 这个题直接逆难度很大，PWN的这些题实际上都是在原来的基础上改编，所以网上一搜befunge就有很多文章，github一搜befunge关键词也有对应的C语言实现，根据这些来进行解题是最佳选择

befunge官网文档 befunge的github一种源码实现

题目分析

  • 一看到这种题，就不要想什么堆风水了，基本不可能，一般都是哪里类型判断不严格，或者负数溢出，或者别的溢出，最后导致越界读和越界写
  • 看到那种可以让用户输入32位数字，或者打印32位数字都要提起注意，漏洞很可能在这里

漏洞在于g（get）和p（put）指令没有check好边界，导致的越界写和越界读：

case 'g':
    {

        int y_ = (int)pop();
        int x_ = (int)pop();

        // vuln here
        push((x_ < rows && y_ < cols) ? code_ptr[x_ * cols + y_] : 0);
    } 
    break;
case 'p':
    {
        int y_ = (int)pop();
        int x_ = (int)pop();
        uint8_t value = (uint8_t)pop();

        // vule here
        if(x_ < rows && y_ < cols){
            code_ptr[x_ * cols + y_] = value; 
        }
    } 
    break;

只需要分配一个很大的内存让其mmap到libc地址的上方，通过越界读写libc里的地址即可

  • 注意0x1F03 add eax, edx指令，超过32溢出的那个1就会被丢弃，详细见exp中的解释

ez_money

前言

  • movdqu xmm0, cs:xmmword_6080+8

指令：movdqu 用途：从内存中加载未对齐的128位数据（可以是整数或浮点数）到 xmm 寄存器。 详细解释： xmm0：目标 SIMD 寄存器。 cs:xmmword_6080+8：从代码段（由 cs 指定）的一个地址开始的位置读取数据。xmmword_6080 是一个标签或地址，+8 表示从这个地址向后偏移8字节。这通常用于跳过某些数据或定位到具体的数据结构内部。

  • movups xmmword ptr [rax+8], xmm0

指令：movups 用途：将 xmm 寄存器中的128位数据存储回未对齐的内存地址。 详细解释： xmmword ptr [rax+8]：目标内存地址，由基址寄存器 rax 加上8字节偏移确定。xmmword ptr 表示目标是一个128位的数据。 xmm0：源 SIMD 寄存器。

题目分析

  • 题目难点主要在于逆向，函数很多，最后的溢出点在于loan_list会溢出，这个vip_chance应当<=10,当>10就会溢出到ck1
  • 因此可以通过这个溢出修改size为一个很大的数同时类似off-by-one的效果把很多个tcache chunk覆盖到unsorted bin chunk中
  • show libcbase和heapbase都是通过I’m vip这个函数，利用unsortedbin 中的值泄露libcbase,利用tcache-key=heapbase+0x10泄露heapbase
  • 正因为off-by-one的效果，可以让account中存在不同的位置存同一个chunk地址
  • 因此这个题没有uaf，libc2.31，还是得有off-by-one或者off-by-null这种方式来实现攻击（这和我对堆攻击的理解一致）
